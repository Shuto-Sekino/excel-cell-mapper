# スキーマDSL仕様

> ステータス: 下書き v0.1

## 概要

スキーマはプレーンなJavaScript/JSONオブジェクトで記述します。
値にセル参照を書くことで、そのセルの値がJSONに展開されます。

---

## セル参照の記法

### 単一セル参照

```
"<列><行>"
```

| 例 | 説明 |
|----|------|
| `"A1"` | A列1行目 |
| `"B3"` | B列3行目 |
| `"AA10"` | AA列10行目 |

### シート指定付きセル参照

```
"<シート名>!<列><行>"
```

| 例 | 説明 |
|----|------|
| `"Sheet2!A1"` | Sheet2のA列1行目 |
| `"顧客情報!B5"` | 「顧客情報」シートのB列5行目 |

### セル範囲参照（配列用）

```
"<開始セル>:<終了セル>"
```

| 例 | 説明 |
|----|------|
| `"A1:A10"` | A列の1〜10行目（縦方向の配列） |
| `"A1:D1"` | A〜D列の1行目（横方向の配列） |
| `"A1:C3"` | A1からC3の2次元範囲 |
| `"Sheet1!A1:A5"` | シート指定付き範囲 |

---

## スキーマ記法

### 1. 静的キー + セル参照（最も基本的な形）

JSONのキー名を文字列で指定し、値にセル参照を指定します。

```json
{
  "name": "B1",
  "age": "B2",
  "email": "B3"
}
```

**出力例:**
```json
{
  "name": "山田太郎",
  "age": 30,
  "email": "yamada@example.com"
}
```

---

### 2. 動的キー（セル参照 → セル参照）

キーと値の両方をセル参照にすると、キーのセルの値をJSONのキー名として使用します。

```json
{
  "A1": "B1",
  "A2": "B2",
  "A3": "B3"
}
```

Excelのデータ:

| | A | B |
|-|---|---|
| 1 | name | 山田太郎 |
| 2 | age | 30 |
| 3 | email | yamada@example.com |

**出力例:**
```json
{
  "name": "山田太郎",
  "age": 30,
  "email": "yamada@example.com"
}
```

---

### 3. ネストしたオブジェクト

スキーマにオブジェクトをネストすることで、階層構造のJSONを生成できます。

```json
{
  "name": "B1",
  "address": {
    "prefecture": "B4",
    "city": "B5",
    "zip": "B6"
  },
  "contact": {
    "email": "B7",
    "phone": "B8"
  }
}
```

**出力例:**
```json
{
  "name": "山田太郎",
  "address": {
    "prefecture": "東京都",
    "city": "渋谷区",
    "zip": "150-0002"
  },
  "contact": {
    "email": "yamada@example.com",
    "phone": "03-1234-5678"
  }
}
```

---

### 4. 配列（セル範囲）

値に範囲参照を配列として指定すると、その範囲の値を配列に変換します。

```json
{
  "items": ["A5:A10"]
}
```

**出力例:**
```json
{
  "items": ["りんご", "みかん", "ぶどう", "もも", "なし", "いちご"]
}
```

> `null` やブランクセルの扱いはオプションで制御可能（後述）

---

### 5. オブジェクト配列（行・列のレンジ）

範囲の各行（または列）を1つのオブジェクトとしてマッピングする記法。

```json
{
  "products": {
    "$range": "A2:C6",
    "$schema": {
      "id": 0,
      "name": 1,
      "price": 2
    }
  }
}
```

> `$range` に対象範囲、`$schema` に各列のインデックス（0始まり）とフィールド名のマッピングを指定します。

**Excelのデータ:**

| A | B | C |
|---|---|---|
| id | name | price |
| 1 | りんご | 100 |
| 2 | みかん | 80 |
| 3 | ぶどう | 200 |

**出力例:**
```json
{
  "products": [
    { "id": 1, "name": "りんご", "price": 100 },
    { "id": 2, "name": "みかん", "price": 80 },
    { "id": 3, "name": "ぶどう", "price": 200 }
  ]
}
```

---

### 6. 複数シートにまたがるマッピング

シート名を含むセル参照を使用することで、異なるシートの値を1つのJSONにまとめられます。

```json
{
  "customer": {
    "name": "顧客情報!B1",
    "id": "顧客情報!B2"
  },
  "order": {
    "date": "注文情報!C1",
    "total": "注文情報!C5"
  }
}
```

---

## 特殊ディレクティブ

スキーマ内で `$` プレフィックスを持つキーは特殊ディレクティブとして扱います。

| ディレクティブ | 説明 |
|---------------|------|
| `$range` | 対象のセル範囲を指定 |
| `$schema` | レンジ内の列/行マッピングを定義 |
| `$direction` | レンジの走査方向 (`"row"` \| `"column"`, デフォルト: `"row"`) |
| `$skipEmpty` | 空白セルの行/列をスキップするか（デフォルト: `false`） |

---

## セル値の型変換

Excelのセル値は以下のルールで自動的にJavaScriptの型に変換されます。

| Excelの型 | JavaScript型 |
|-----------|-------------|
| 数値 | `number` |
| 文字列 | `string` |
| 日付 | `Date`（オプションで文字列形式も可） |
| 真偽値 | `boolean` |
| 空白セル | `null`（オプションで `undefined` や省略も可） |
| エラー値 | `null`（またはエラーハンドラで処理） |
